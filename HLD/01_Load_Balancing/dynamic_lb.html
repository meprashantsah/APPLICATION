Dynamic Load Balancing

Algorithms that consider the real-time state of servers — active connections, response time, CPU load — to make smarter routing decisions.

1. Least Connections

Routes the new request to the server with the fewest active connections. This assumes all requests take roughly the same time to process.

🔑 Key Idea A server with fewer active connections is likely less busy. Send the new request there.

Example Walkthrough

Initial state: S1=2 connections, S2=5 connections, S3=1 connection.

StepServer StatesNew Request Goes ToReason
StartS1=2, S2=5, S3=1—Initial state
Req 1S1=2, S2=5, S3=1S3Lowest connections (1)
AfterS1=2, S2=5, S3=2—S3 now has 2
Req 2S1=2, S2=5, S3=2S1Tie at 2, pick first (or round-robin tie-break)
AfterS1=3, S2=5, S3=2—S1 now has 3
Req 3S1=3, S2=5, S3=2S3Lowest connections (2)
import java.util.HashMap;
import java.util.Map;

public class LeastConnectionsLoadBalancer {
    private final Map servers = new HashMap<>();

    public LeastConnectionsLoadBalancer() {
        servers.put("S1", 2);
        servers.put("S2", 5);
        servers.put("S3", 1);
    }

    public synchronized String leastConnections() {
        String bestServer = null;
        int minConnections = Integer.MAX_VALUE;

        for (Map.Entry entry : servers.entrySet()) {
            if (entry.getValue() < minConnections) {
                minConnections = entry.getValue();
                bestServer = entry.getKey();
            }
        }
        return bestServer;
    }

    public void routeRequest() {
        String server = leastConnections();
        servers.put(server, servers.get(server) + 1);
        System.out.println("Request routed to " + server);
    }

    public static void main(String[] args) {
        LeastConnectionsLoadBalancer lb = new LeastConnectionsLoadBalancer();
        lb.routeRequest();  // S3
    }
}

2. Weighted Least Connections

Combines Least Connections with server weights. A powerful server can handle more connections before being considered "full."

Formula: Score = Active Connections / Weight

The server with the lowest score gets the request.

ServerConnectionsWeightScoreDecision
S1636/3 = 2.0✅ Pick this (lowest)
S2424/2 = 2.0Tie with S1
S3313/1 = 3.0Higher score
⚠️ Tie-Breaking When scores tie (like S1 and S2 above), you can use Round Robin as a tie-breaker, or simply pick the first one in the list.

3. Least Response Time

Routes to the server with the lowest average response time. This is the most accurate measure of server health but requires constant monitoring. 

import java.util.HashMap;
import java.util.Map;

public class LeastResponseTimeLoadBalancer {
    private final Map servers = new HashMap<>();

    public LeastResponseTimeLoadBalancer() {
        servers.put("S1", new ServerInfo(45, 10));
        servers.put("S2", new ServerInfo(120, 8));
        servers.put("S3", new ServerInfo(30, 15));
    }

    public String leastResponseTime() {
        String bestServer = null;
        int minResponseTime = Integer.MAX_VALUE;

        for (Map.Entry entry : servers.entrySet()) {
            if (entry.getValue().avgResponseMs < minResponseTime) {
                minResponseTime = entry.getValue().avgResponseMs;
                bestServer = entry.getKey();
            }
        }
        return bestServer;
    }

    public void routeRequest() {
        String server = leastResponseTime();
        System.out.println("Request routed to " + server + " (" + servers.get(server).avgResponseMs + "ms)");
    }

    public static void main(String[] args) {
        LeastResponseTimeLoadBalancer lb = new LeastResponseTimeLoadBalancer();
        lb.routeRequest();  // S3 (30ms)
    }

    static class ServerInfo {
        int avgResponseMs;
        int connections;

        ServerInfo(int avgResponseMs, int connections) {
            this.avgResponseMs = avgResponseMs;
            this.connections = connections;
        }
    }
}
💡 Why Response Time Matters A server might have few connections but each connection is doing heavy work (e.g., complex DB queries). Response time captures this better than connection count alone.

4. Resource-Based (Agent-Based)

An agent running on each server reports real-time metrics (CPU, memory, disk I/O) to the load balancer. The LB routes based on which server has the most free resources.

✅ When to Use Dynamic LB
  • Requests have varying processing times (some fast, some slow)
  • Traffic patterns are unpredictable
  • Servers have different capacities
  • You need to handle traffic spikes gracefully
❌ Downsides
  • More complex to implement and debug
  • Requires health checks and monitoring overhead
  • Response time measurement adds latency
  • Can be "too reactive" — oscillates between servers
📝 Interview Tip "In production, I would start with Least Connections for most web applications. For compute-heavy workloads (like ML inference), I'd use Resource-Based load balancing because CPU is the real bottleneck, not connection count."
